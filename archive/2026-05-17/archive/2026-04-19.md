# 2026-04-19

## Ingestion Spec Snapshot

Created a dedicated ingestion pipeline spec in `docs/ingestion-pipeline-spec.md`.

Key decisions captured there:
- ingestion v1 is deterministic and rule-based
- the pipeline stops at `ResourceCandidate`, not final `Resource`
- raw Twitter bookmark payloads remain durable evidence
- nested URL signals from the raw export are first-class ingestion inputs
- media and article-title-only artifacts are preserved for later review instead of guessed aggressively
- reimports create new `ImportRun` rows but upsert artifacts by connector artifact id
- the first required CLI contract is `pile import twitter <path>`
