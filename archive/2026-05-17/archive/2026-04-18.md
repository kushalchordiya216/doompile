# 2026-04-18

## Brainstorming Summary

Today clarified that the product is not a generic bookmark archive or a markdown mirror.

The real goal is a CLI-first learning planner built on top of saved resources, with Twitter bookmarks as the first input source. The problem is not lack of resources; it is lack of sequencing, prioritization, and follow-through.

The desired user experience became:
- ask to learn a topic or goal
- get an opinionated learning path
- see concepts in order
- get a few strong resources per step
- move from foundations to hands-on work
- track progress over time

We decided that the product should optimize for committed study rather than broad browsing.

We also clarified that screenshots and media are usually not the final resource. They are evidence that should help recover the canonical source. OCR is useful when it helps trace back to the real article, playlist, video series, paper, or repo.

## Key Product Decisions

- Product framing: goal-first, source-first, reviewable, learning-oriented
- Main mode: committed study
- Output shape: proposed learning path, not a raw ranked list
- Curation style: high quality with manual review
- Search is useful, but not the product center
- Graph-shaped data model is a better fit than pure semantic retrieval
- Twitter bookmarks should be emphasized in product positioning, but the architecture must remain connector-generic
- The future tutor is downstream of this storage and planning layer, not part of v1

## Naming Decisions

- Product name: `Doompile`
- CLI command: `pile`

The name leans into the bookmark-hoarder angle while still being memorable and marketable on Twitter.

## Technical Decisions

- Language: Python
- Storage: SQLite
- CLI-first interface
- Thin dashboard later
- PydanticAI may be used behind a local `ModelRouter`
- Langfuse is the default tracing and evals backend
- No LiteLLM
- No LangChain or LangGraph as the core architecture
- Strong typing, explicit interfaces, structured logging, and observability are first-class concerns

## Architecture Direction

Planned layers:
- connectors
- ingestion
- resolution
- review
- resources
- concepts
- planning
- progress
- ai
- tracing

Important architectural rule:
- external frameworks may help at the edges, but they should not own the system design

## Corpus Learnings

From the current Twitter bookmarks export:
- 340 bookmarks total
- 234 contain `t.co` links in text
- 187 contain media
- about 154 appear recoverable without OCR
- about 128 likely need media-based recovery
- about 58 likely need text-only inference or manual review

Important takeaway:
- the raw `bookmarks.json` contains useful nested source signals that do not survive in the simplified filtered export
